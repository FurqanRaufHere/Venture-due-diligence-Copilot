Executive Summary LLM
Financial Analysis DETERMINISTIC
ANOMALY EXPLANATION — LLM
Financial metrics are computed deterministically using Python (Pandas/NumPy) — no LLM involved in the numbers themselves. Revenue CAGR uses the compound annual growth rate formula over all available years. Burn rate is the average monthly net cash outflow. Runway = current cash balance ÷ monthly burn. The LLM only writes the explanation text above, grounded in the actual computed numbers.
Extracted Claims CLAIM EXTRACTION AGENT
Claims are extracted using a structured JSON schema enforced prompt sent to the LLM. The model is instructed to return only valid JSON — no preamble or explanation. Temperature is set to 0.1 (near-deterministic) to minimize hallucination. The output is validated with Pydantic before being stored. If the LLM fails to return valid JSON, the extraction agent returns empty fields gracefully without crashing the pipeline.
Market & Competition RAG AGENT
MARKET ASSESSMENT — LLM-GENERATED FROM RETRIEVED DATA
RAG (Retrieval-Augmented Generation) means the LLM's market assessment is grounded in retrieved data, not hallucinated. The process: (1) The startup's solution + target market are embedded into a 384-dimensional vector. (2) FAISS (vector database) retrieves the most similar companies from our 74-startup dataset. (3) Those retrieved companies are injected into the LLM prompt as context. (4) The LLM writes the assessment based on that context — not from memory. This prevents the LLM from making up competitor names.
Founder Risk Profile FOUNDER AGENT
POSITIVE SIGNALS
RISK FLAGS
Credibility scoring is hybrid: (1) The LLM extracts structured facts from the bio text (prior exits, years experience, domain relevance, role gaps) using a strict JSON schema. (2) Rule-based logic then assigns points based on those facts — prior exits add 20 points, 10+ years domain experience adds 15 points, missing technical founder deducts 10 points, etc. (3) The LLM writes the explanation. This prevents the LLM from generating an arbitrary "vibe-based" score.
Failure Pattern Analysis FAISS SIMILARITY
SIMILAR FAILED STARTUPS
SIMILAR SUCCESSFUL STARTUPS
Pattern matching uses sentence-transformer embeddings (all-MiniLM-L6-v2, 384 dimensions) and FAISS vector search. The startup description is embedded into a vector and compared against 74 pre-embedded historical startups using cosine similarity. Similarity scores above 0.45 are considered meaningful matches. The dominant failure archetype is determined by the most common failure category among high-similarity (>55%) failed matches.
